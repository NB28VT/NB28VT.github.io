Current Projects

Fantasy Fishbowl

Fantasy Fishbowl is a fantasy sports app with a twist. Instead of predicting sports performance, players try and guess which songs the Vermont rock band Phish will play ahead of a given concert.

Before each performance, players in a league attempt to guess which song the band will open with, which song will end the first set, etc. Players earn points for correctly predicting a song that's played, and extra points if they correctly guess the setlist placement. It's loosely based on a pen and paper game me and my friends play before their concerts.

Fantasy Fishbowl has always been my major personal project, and it's constantly evolved and been overhauled as my skills as a software developer have improved. The current iteration is by far the most complete, and I hope to have all major features working in 2020.

Current Features:

  • Mobile-first web interface for viewing upcoming concerts, making predictions, and comparing your score to your friends.
  • Automatic scoring using setlists published after a concert via the Phish.net fan API.
  • An API-only backend built with Rails 5.
  • A React.js front-end written in TypeScript.
  • JSON Web Token User Authentication.

For more information on some of the specific technologies I used, check out the frontend source code on GitHub here. The backend source code is here.

MBTA Live Departure Board

I'm building this single-page application as a real-time representation of the train departure board at Boston's South Station terminal. This project is built in React.js and utilizes the powerful Massachusetts Bay Transit Authority developer API and its streaming service to consistently update train departure predictions.

Current Features:

  • Predicted departure times, train destinations, train numbers and the status of each trip ("on time", "delayed" etc.)
  • Live updates whenever the MBTA server sends out a "reset" event.
  • A simple live clock.

You can check it out for yourself here; the source code is on GitHub here.

Past Projects

"Who's In It?" The Movie cast guessing game (2017)

"Who's In It?"" is a movie cast matching game I built while teaching myself React.js in 2017. Players search for a movie by title, select from the search results, and see a group of five cast member headshots on the gameboard. They must match each image to a scrambled list of names to win the game.

React has fast become one of my favorite frameworks to work with, and building an interactive game that loads data from an API was a perfect challenge for learning the basics of the framework. Movie data is retrieved from The Movie Database API. You can play the game here.

F1 racing stats API: Ruby Gem wrapper (2017)

Formula One racing has never been very popular in the United States, but it's been one of my nerdier interests since I was a little kid, going up to Montreal to see the Canadian Grand Prix. I love the technology and engineering challenge the teams are faced with, and I know way too much about the sport's history. So when I discovered the Ergast F1 stats API, I knew I had to use it in a project.

Working with APIs was one of the first aspects of software development I really enjoyed, and nobody had written a Ruby wrapper for the Ergast API at the time, so I decided to build my own as a Ruby Gem package. As of Spring 2020, it's been downloaded over 650 times.

The source code is hosted on GitHub here.

About Me
